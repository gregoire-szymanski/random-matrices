# Project Guide

`mpdiff` is a config-driven numerical toolkit for high-dimensional diffusion simulation and Marcenko-Pastur (MP) spectral analysis.

This document is the central entry point for the codebase. It is designed so a new research engineer can run experiments and interpret outputs without reading implementation details first.

## A. Project Overview

### What the project does

The project supports the full workflow:

1. simulate a `d`-dimensional diffusion with piecewise-constant volatility,
2. compute realized covariance from simulated increments,
3. estimate empirical spectral density from realized eigenvalues,
4. run MP forward transform from a population law,
5. run MP inverse methods to recover a population law from an observed law,
6. compare recovered and reference population spectra.

### Key features

- Reproducible experiments via config seeds.
- Multiple covariance/volatility model families:
  - isotropic diagonal,
  - diagonal with sampled eigenvalues,
  - rotated covariance `U diag(lambda) U^T`,
  - low-rank plus diagonal `B Sigma B^T + D`,
  - piecewise and piecewise-scaled-base regimes.
- Multiple MP inverse methods behind one interface.
- Method comparison mode (`compare_methods` and `compare_all_methods`).
- End-to-end runners with saved artifacts (arrays, plots, CSV/JSON summaries, reports).
- Unit/integration tests for simulation, spectral transforms, inversion, configs, CLI parsing, and output contracts.

### Typical workflows

- Simulation workflow:
  - `config -> volatility schedule -> path simulation -> realized covariance -> empirical spectrum`
- Spectral workflow:
  - `population law H -> MP forward F`
- Inverse workflow:
  - `observed density F -> inverse method -> estimated H_hat`
- End-to-end validation:
  - `simulate -> empirical F_emp -> inverse -> compare H_hat vs reference H_ref`

### Component interaction map

```text
config loader/validator
        |
        v
volatility schedule builder ----> covariance builders ----> random-matrix helpers
        |
        v
diffusion simulator ----> realized covariance/eigenvalues ----> empirical spectral density
                                                              |
                                                              v
population law builders ----> MP forward transform ----> MP inverse methods
                                                              |
                                                              v
                                             metrics + plotting + artifact writers
```

## C. Key Concepts Mapping

### Population spectral law (`H`)

In code, the population law is represented as:

- `DiscreteSpectrum` (atoms + weights),
- `ParametricSpectrumLaw` converted to discrete atoms,
- `GridDensity` converted to discrete atoms.

Core location: `src/mpdiff/spectral/densities.py`.

Interpretation by model:

- Constant covariance model: eigenvalue law of the model covariance matrix.
- Piecewise model in full pipeline: reference is eigenvalue law of the *integrated covariance* `bar(Sigma) = (1/T) * integral_0^T Sigma_t dt`.

### Empirical spectral density (`F_emp`)

Derived from realized covariance eigenvalues:

1. simulate path `X_{t_i}`,
2. increments `Delta X_i = X_{t_{i+1}} - X_{t_i}`,
3. realized covariance `RCV = (Delta X^T Delta X) / scale`,
4. eigenvalues of `RCV`,
5. KDE or histogram on a grid.

Core location: `src/mpdiff/spectral/empirical.py`.

### Realized covariance and MP connection

For high-dimensional asymptotics with aspect ratio `c = d/n`, the empirical spectrum of covariance-type estimators is linked to population spectrum through MP transform.

In the code:

- Forward: `H -> F = MP_c(H)` (`spectral/transforms.py`).
- Inverse: `F -> H_hat` approximately (`spectral/inverse.py` + `inversion_methods/`).

### Covariance model mapping

- `diag_scalar`: `Sigma = lambda I_d`.
- `diag_distribution`: `Sigma = diag(lambda_1, ..., lambda_d)` with sampled eigenvalues.
- `orthogonal_diag`: `Sigma = U diag(lambda) U^T`.
- `low_rank_plus_diag`: `Sigma = B diag(s) B^T + D`.
- `piecewise`: time-segmented `Sigma_t`.
- `piecewise_scaled_base`: `Sigma_j = c_j M_j` with shared or redrawn base matrices.

## D. How To Run The Project

### 1. Setup

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e .[dev]
```

Check CLI availability:

```bash
mpdiff --help
```

### 2. Run simulation only

```bash
mpdiff simulation --config configs/config_constant_isotropic.yaml
```

This writes path arrays, realized covariance artifacts, and optional figures to `global.output_dir`.

### 3. Run MP forward transform

```bash
mpdiff mp-forward --config configs/config_diagonal_gamma.yaml
```

Expected key artifacts:

- `mp_forward_density.npz`
- `population_density.npz`
- `mp_forward_metadata.json`
- `mp_forward_comparison.png` (if figure saving enabled)

### 4. Run MP inverse (synthetic)

```bash
mpdiff mp-inverse --config configs/config_diagonal_dirac_mixture.yaml
```

Switch methods in config:

```yaml
mp_inverse:
  method: optimization
  compare_methods: [optimization, fixed_point, moment_based]
```

Or run all registered methods:

```yaml
mp_inverse:
  method: optimization
  compare_all_methods: true
```

### 5. Run full end-to-end pipeline

```bash
mpdiff full-pipeline --config configs/config_piecewise_two_segments.yaml
```

Alias:

```bash
mpdiff end-to-end --config configs/config_piecewise_two_segments.yaml
```

### 6. Run inversion benchmark runner

```bash
mpdiff inversion-benchmark --config configs/config_diagonal_gamma.yaml
```

This compares inversion methods on a shared benchmark law and writes runtime/quality tables.

## E. Config System Explanation

Config files are YAML/TOML parsed into `ProjectConfig` dataclasses.

Top-level sections:

```yaml
global:
simulation:
volatility:
mp_forward:
mp_inverse:
analysis:
plotting:
benchmark:
```

### Section summary

- `global`: seed, output directory, save switches, logging level.
- `simulation`: `d`, horizon `T`, `n_steps`, initial state, drift model.
- `volatility`: covariance model family and segment policies.
- `mp_forward`: forward solver grid and fixed-point controls.
- `mp_inverse`: inversion method and numerical controls.
- `analysis`: normalization, histogram bins, optional population spectrum overrides.
- `plotting`: style, figure size, plot toggles.
- `benchmark`: timer logging enable/disable.

### Annotated minimal example

```yaml
global:
  seed: 20260408
  output_dir: outputs/example_run
  log_level: INFO
  save_figures: true
  save_arrays: true
  save_metadata: true

simulation:
  d: 80
  T: 1.0
  n_steps: 400
  initial_state: 0.0
  drift_model:
    kind: zero

volatility:
  mode: constant
  constant_model:
    kind: diag_scalar      # Cov = lambda I_d
    scalar: 1.2

mp_forward:
  aspect_ratio: 0.2        # c = d/n
  grid_min: 0.0
  grid_max: 5.0
  num_points: 500
  eta: 0.003
  tol: 1.0e-9
  max_iter: 500
  damping: 0.72

mp_inverse:
  method: optimization
  compare_methods: [optimization, fixed_point, moment_based]
  n_support: 40
  support_min: 0.05
  support_max: 3.5
  regularization: 1.0e-3

analysis:
  realized_covariance_normalization: total_time
  empirical_histogram_bins: 50

plotting:
  show: false

benchmark:
  enabled: true
```

### Piecewise example skeleton

```yaml
volatility:
  mode: piecewise
  segments:
    - start: 0.0
      end: 0.4
      scalar: 1.0
      model:
        kind: diag_distribution
        eigen_distribution:
          kind: gamma
          shape: 2.0
          scale: 0.3
    - start: 0.4
      end: 1.0
      scalar: 1.2
      model:
        kind: orthogonal_diag
        eigen_distribution:
          kind: uniform
          low: 0.4
          high: 1.4
        orthogonal:
          method: haar
```

Validation rules enforce contiguous segments and end time equal to `simulation.T`.

## F. Tests

### Organization

Tests mix unit and lightweight integration checks:

- unit: spectral math, covariance builders, schedule logic,
- integration: runner output file contracts,
- config: schema/validation and catalog examples,
- interface: CLI parser and notebook config references.

### Run tests

Run all:

```bash
pytest
```

Run one file:

```bash
pytest tests/test_mp_inverse.py
```

Run by keyword:

```bash
pytest -k "forward and inverse"
```

Verbose mode:

```bash
pytest -v
```

Run one specific test:

```bash
pytest tests/test_mp_forward.py::test_mp_forward_dirac_matches_closed_form_reasonably -v
```

Coverage (optional):

```bash
pip install pytest-cov
pytest --cov=mpdiff --cov-report=term-missing
```

Expected runtime:

- Typical laptop local run: usually under ~30 seconds.
- Heavier environments may vary due to BLAS/matplotlib/font-cache startup.

### What a passing numerical test means

Passing does not mean exact equality to theory at machine precision.

Tests check robust numerical behavior under discretization/regularization:

- tolerance bands on relative errors,
- moment proximity,
- non-negativity and finite outputs,
- consistency of produced artifacts.

Numerical transforms are regularized (`eta > 0`) and finite-grid approximated, so moderate tolerances are intentional.

All outputs go to `global.output_dir`.

- `.npy`: arrays (paths, covariances, eigenvalues, atoms, weights),
- `.npz`: grid densities (`grid`, `density`),
- `.csv`/`.json`: summary tables,
- `.txt`: report text,

- `simulation`: `times.npy`, `paths.npy`, `realized_covariance.npy`, `realized_density.npz`, `metadata.json`, optional plots.

1. check `*_summary.csv/json` for metric ranking,
2. inspect overlay plots,

4. verify method agreement before drawing conclusions.

## J. Performance And Timers

Timers are recorded with `timed_block` and optionally logged when `benchmark.enabled: true`.

Typical timed stages:

- simulation setup and trajectory generation,
- realized covariance and eigendecomposition,
- empirical density estimation,
- MP forward/inverse core loops,
- benchmark setup and method loops.

Where to inspect:

- logs during execution,
- metadata JSON files under `timers_seconds`.

Most expensive blocks in typical workloads:

- MP inverse optimization loops,
- repeated MP forward evaluations inside inverse methods,
- large-grid KDE/eigendecomposition for high dimensions.

Practical speed tips:

- reduce `mp_forward.num_points` for early experimentation,
- lower `mp_inverse.n_support` when debugging,
- start with fewer `compare_methods`,
- tune `optimizer_max_iter` and solver tolerances for iteration speed.
